服务端和客户端的逻辑基本一样
首先在主线程启动socket监听(StartServerListen),等待用户连接(死循环);如果有用户连接,说明是客户发数据过来,这在该线程再启动接收数据线程(RevcSocketData)
然后在主线程启动读用户输入数据(ReadSystemIn 死循环);如果用户在控制台输入数据,则在该线程启动发送数据线程(SendSocketData)


也就是说程序总共有五个线程
一个主线程
一个监听线程
一个等待用户输入线程
一个发送数据线程(发送完数据,该线程死亡,之后又数据发送,则重新new出一个接受线程)
一个接受数据线程(接受完数据,该线程死亡,之后又有数据接收,则重新new出一个发送线程)
